## Closed Gaps

- Story Agent browser runtime no longer sends model requests directly to `api.deepseek.com`.
- Provider credentials are resolved server-side by the model gateway.
- Imported worldbook and preset macros no longer leak raw `{{...}}` ST syntax into the provider prompt.
- Status UI remains renderable even when the model omits the status JSON block; the fallback source tag is generated by the Story runtime and stripped by the UI renderer.

## Remaining Gaps

- The status fallback is intentionally minimal (`待更新`) when the model omits structured status data. A later pass should improve extraction from generated timeline text and StoryState memory.
- Prompt topology is still influenced by deliberate assistant/user exemplars from the imported preset. This is a preset-compiler policy question, not a transport issue.
