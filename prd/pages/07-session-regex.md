# 对话工作台 - 正则脚本编辑器

> **路由：** `/session` (regex 视图)
> **模块：** 对话
> **组件：** `RegexScriptEditor` → `HeaderBar` + `Toolbar` + `SourceTabs` + `ScriptListItem`

## 概述

正则脚本编辑器用于管理文本后处理规则。当 LLM 生成回复后，系统会按顺序执行所有启用的正则脚本，对输出文本进行替换、格式化、过滤等处理。这是实现角色卡高级显示效果（如彩色文本、自定义格式、敏感词过滤）的关键工具。

## 字段

### 脚本列表

| 列名 | 格式 | 可排序 | 说明 |
|------|------|--------|------|
| 展开/折叠 | 箭头 | 否 | 展开查看脚本详情 |
| 脚本名称 | 文本 | 是 | 脚本的描述性名称 |
| 来源 | 标签 | 是 | 角色级 / 全局级 |
| 启用 | 开关 | 否 | 是否在后处理中执行 |
| 操作 | 按钮组 | 否 | 编辑、删除、排序 |

### 正则脚本编辑器（RegexScriptEntryEditor）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 脚本名称 | 文本输入 | 是 | 脚本标识名称 |
| 查找模式 | 文本输入 | 是 | 正则表达式模式 |
| 替换文本 | 文本输入 | 是 | 替换内容（支持 `$1` 等捕获组引用） |
| 应用目标 | 多选 | 否 | 应用到哪些内容（AI 输出 / 用户输入 / 显示文本 / 提示词） |
| 匹配标志 | 多选 | 否 | 正则标志（g / i / m / s） |
| 启用 | 开关 | 否 | 默认启用 |
| 最小深度 | 数字 | 否 | 仅对深度 ≥ N 的消息生效 |
| 最大深度 | 数字 | 否 | 仅对深度 ≤ N 的消息生效 |

## 交互

### 页面加载
- 加载当前角色绑定的正则脚本和全局正则脚本
- 在列表中分组显示

### 来源筛选
- **触发：** 切换来源标签页（全部 / 角色 / 全局）
- **行为：** 过滤显示对应来源的脚本

### 新建脚本
- **触发：** 点击"新建脚本"
- **弹窗：** 打开空白正则脚本编辑器
- **保存：** 脚本添加到列表

### 编辑脚本
- **触发：** 点击脚本行的"编辑"按钮
- **弹窗：** 打开预填数据的编辑器

### 删除脚本
- **触发：** 点击"删除"
- **确认：** 二次确认后移除

### 启用/禁用
- **触发：** 切换启用开关
- **行为：** 禁用的脚本不参与后处理

### 调整执行顺序
- **触发：** 点击上移/下移按钮
- **行为：** 正则脚本按列表顺序依次执行，顺序影响最终效果

### 导入正则脚本
- **触发：** 点击"导入"
- **弹窗：** ImportRegexScriptModal
- **格式：** SillyTavern 正则脚本 JSON

### 导出正则脚本
- **触发：** 点击"导出"
- **行为：** 导出为 JSON 文件

### 正则调试器
- **触发：** 点击"调试器"按钮
- **面板：** RegexDebuggerPanel
- **功能：** 输入测试文本，实时查看正则匹配和替换结果

## API 依赖

| 操作 | 来源 | 触发时机 | 说明 |
|------|------|---------|------|
| 加载脚本 | `function/regex/get` | 视图切换 | 获取角色和全局正则脚本 |
| 添加脚本 | `function/regex/add` | 新建保存 | 创建正则脚本 |
| 更新脚本 | `function/regex/update` | 编辑保存 | 更新脚本内容 |
| 删除脚本 | `function/regex/delete` | 删除确认 | 移除脚本 |
| 导入 | `function/regex/import` | 导入确认 | 解析并存储导入脚本 |
| 全局操作 | `function/regex/global` | 全局管理 | 管理全局正则脚本 |

## 页面关系

- **入口：** 对话工作台视图切换按钮
- **出口：** 视图切换回聊天/世界书/预设
- **数据耦合：** 正则脚本在对话生成后通过 RegexNode 自动执行后处理

## 业务规则

- 正则脚本在 NodeFlow 管道的 RegexNode 阶段执行
- 执行顺序：按列表排列顺序依次执行，前一个脚本的输出是后一个脚本的输入
- 应用目标决定脚本作用于哪些内容流：
  - AI 输出：LLM 生成的回复文本
  - 用户输入：用户发送的消息
  - 显示文本：渲染前的最终文本
  - 提示词：发送给 LLM 的提示词
- 深度范围限制脚本仅对特定位置的消息生效
- 替换文本支持正则捕获组引用（`$1`、`$2` 等）
- 正则脚本数据格式兼容 SillyTavern 正则脚本规范
