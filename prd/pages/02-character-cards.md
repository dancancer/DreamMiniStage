# 角色卡库

> **路由：** `/character-cards`
> **模块：** 角色管理
> **组件：** `CharacterCards`

## 概述

角色卡管理中心，用户在此浏览、导入、编辑和管理所有可用的角色卡。角色卡是 SillyTavern 兼容的 PNG 格式文件，嵌入了角色的名称、描述、人设、场景、开场白等结构化数据。用户可以从角色卡库选择角色来创建新的对话会话。

## 字段

### 角色卡片

| 字段 | 类型 | 说明 |
|------|------|------|
| 角色头像 | 图片 | 嵌入在 PNG 角色卡中的头像图 |
| 角色名称 | 文本 | 角色的显示名称 |
| 操作按钮 | 按钮组 | 编辑、删除、置顶（悬停/长按显示） |

### 角色编辑弹窗（EditCharacterModal）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | 文本输入 | 是 | 角色显示名称 |
| 描述 | 多行文本 | 否 | 角色外貌、特征描述 |
| 人设 | 多行文本 | 否 | 角色性格、行为模式 |
| 场景 | 多行文本 | 否 | 对话发生的场景设定 |
| 开场白 | 多行文本 | 否 | 角色的第一条消息（支持多条） |
| 头像 | 图片上传 | 否 | 角色头像图片 |

### 角色导入弹窗（ImportCharacterModal）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 文件选择 | 文件拖放区 | 是 | 支持 PNG 格式角色卡文件 |

## 交互

### 页面加载
- 调用 `getAllCharacters()` 获取所有已导入角色
- 无角色时显示空状态引导
- 桌面端默认显示网格视图，可切换为轮播视图
- 移动端仅显示网格视图

### 两种运行模式

**浏览模式（默认）：**
- 正常浏览、编辑、删除角色卡

**创建会话模式：**
- 从首页"新冒险"按钮进入时激活
- 点击任意角色卡 → 自动创建会话 → 导航到 `/session?id={newSessionId}`

### 导入角色
- **触发：** 点击"导入角色"按钮
- **弹窗：** 打开文件拖放区域
- **支持格式：** PNG（SillyTavern 角色卡格式，元数据嵌入在 PNG chunk 中）
- **成功：** 角色出现在列表中，Toast 通知
- **失败：** 显示错误信息（如格式不支持、数据损坏）

### 编辑角色
- **触发：** 点击角色卡的编辑按钮
- **弹窗：** 打开角色编辑弹窗，预填现有数据
- **保存：** 更新角色数据，刷新卡片显示
- **取消：** 关闭弹窗，不保存

### 删除角色
- **触发：** 点击角色卡的删除按钮
- **确认：** 二次确认对话框
- **成功：** 角色从列表移除
- **注意：** 不会删除已经基于该角色创建的会话

### 置顶角色
- **触发：** 点击角色卡的置顶按钮
- **行为：** 将角色移至列表第一位
- **调用：** `moveToTop(characterId)`

### 视图切换
- **触发：** 点击视图切换按钮（仅桌面端）
- **选项：** 网格（Grid）/ 轮播（Carousel）
- **轮播模式：** 角色卡以 3D 倾斜效果旋转展示（react-parallax-tilt），尊重 `prefers-reduced-motion`

## API 依赖

| 操作 | 来源 | 触发时机 | 说明 |
|------|------|---------|------|
| `getAllCharacters()` | `function/character/list` | 页面加载 | 获取所有角色卡数据 |
| `deleteCharacter(id)` | `function/character/delete` | 删除确认 | 删除指定角色 |
| `moveToTop(id)` | `function/character/move-to-top` | 点击置顶 | 修改角色排序 |
| 导入角色 | `function/character/import` | 文件上传完成 | 解析 PNG 角色卡并存储 |
| `createSession()` | `useSessionStore` | 创建会话模式下点击角色 | 创建新会话 |

## 页面关系

- **入口：**
  - 首页左导航"角色卡"
  - 首页"新冒险"（进入创建会话模式）
- **出口：**
  - 创建会话模式点击角色 → `/session?id={id}`
  - 左导航切换其他页面
- **数据耦合：** 角色卡数据与 `useSessionStore` 在创建会话时交互；角色修改不会影响已创建会话中的角色副本

## 业务规则

- 角色卡采用 SillyTavern V2 规范，元数据嵌入 PNG tEXt chunk
- 支持的角色卡字段：name、description、personality、scenario、first_mes、mes_example、system_prompt、post_history_instructions、tags、creator、character_version
- 导入时自动解析 PNG 内嵌元数据
- 角色卡数据存储在 IndexedDB 中
- 一个角色可以关联多个会话
