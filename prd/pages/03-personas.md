# 人格管理

> **路由：** `/personas`
> **模块：** 人格系统
> **组件：** `PersonasPage`

## 概述

用户人格管理页面，允许用户创建和管理多个"人格"身份。人格代表用户在对话中扮演的角色（而非 AI 角色），包含名称、描述、头像等信息。不同会话可以绑定不同的人格，实现多身份切换。

## 字段

### 人格卡片

| 字段 | 类型 | 说明 |
|------|------|------|
| 头像 | 图片 | 用户上传或默认首字母缩写 |
| 名称 | 文本 | 人格显示名称 |
| 描述 | 文本 | 人格简要描述（截断显示） |
| 默认标记 | 星形图标 | 标识当前默认人格 |
| 操作按钮 | 按钮组 | 编辑、删除、设为默认、激活（悬停显示） |

### 人格编辑器（PersonaEditor）

| 字段 | 类型 | 必填 | 默认值 | 验证 | 说明 |
|------|------|------|--------|------|------|
| 名称 | 文本输入 | 是 | — | 不能为空 | 人格显示名称 |
| 描述 | 多行文本 | 否 | — | — | 人格的自我介绍/背景描述 |
| 注入位置 | 下拉选择 | 否 | 默认位置 | — | 人格描述注入到提示词的位置 |
| 注入深度 | 数字输入 | 否 | 默认深度 | 正整数 | 人格描述在消息历史中的注入深度 |
| 角色 | 下拉选择 | 否 | system | — | 注入消息的角色（system/user/assistant） |
| 头像 | 图片上传 | 否 | 首字母缩写 | — | 自定义头像图片 |

## 交互

### 页面加载
- 从 `usePersonas()` 加载所有已创建人格
- 无人格时显示空状态引导
- 默认人格卡片带有星形标记

### 创建人格
- **触发：** 点击"创建新人格"按钮
- **弹窗：** 打开 PersonaEditor 编辑器（空白表单）
- **验证：** 名称不能为空
- **成功：** 新人格出现在网格中，Toast 通知
- **失败：** 表单内显示验证错误

### 编辑人格
- **触发：** 点击人格卡片的"编辑"按钮
- **弹窗：** 打开 PersonaEditor，预填现有数据
- **保存：** 更新人格数据，刷新卡片
- **取消：** 关闭弹窗

### 删除人格
- **触发：** 点击人格卡片的"删除"按钮
- **确认：** AlertDialog 二次确认
- **成功：** 人格从列表移除
- **注意：** 如果删除的是默认人格，系统不会自动设置新的默认人格

### 设为默认
- **触发：** 点击"设为默认"按钮
- **行为：** 该人格成为全局默认人格，其他人格取消默认标记
- **影响：** 新会话自动使用默认人格（除非角色卡绑定了特定人格）

### 激活人格
- **触发：** 点击"激活"按钮
- **行为：** 立即将该人格设为当前活跃人格（临时，不改变默认设置）

### 导出全部
- **触发：** 点击"导出全部"按钮
- **行为：** 将所有人格数据导出为文件

## API 依赖

| 操作 | 来源 | 触发时机 | 说明 |
|------|------|---------|------|
| 加载人格列表 | `usePersonas()` / Zustand Store | 页面加载 | 从持久化存储读取 |
| 创建人格 | `createPersona()` | 编辑器保存 | 写入 Zustand + 持久化 |
| 更新人格 | `updatePersona()` | 编辑器保存 | 更新指定人格字段 |
| 删除人格 | `deletePersona()` | 删除确认 | 移除人格记录 |
| 设置默认 | `setAsDefault()` | 点击设为默认 | 更新默认标记 |
| 激活人格 | `activatePersona()` | 点击激活 | 设置当前活跃人格 |

## 页面关系

- **入口：** 左侧导航栏"人格"入口
- **出口：** 通过左导航切换到其他页面
- **数据耦合：** 
  - `usePersonaStore` 全局共享人格状态
  - 对话工作台中的 `useCurrentPersona` 消费人格数据
  - 人格变更实时影响对话中的用户身份标识

## 业务规则

- 人格数据通过 Zustand persist 中间件持久化到 localStorage
- 人格名称用作对话中的用户身份显示
- 人格描述会被注入到 LLM 提示词中（根据注入位置和深度配置）
- 支持头像上传，无头像时自动生成首字母缩写图标
- 人格系统独立于认证系统 — 访客模式也可以创建和使用人格
